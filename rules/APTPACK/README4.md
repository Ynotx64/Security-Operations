# APTPACK

## Purpose
This branch contains the organized multi-phase APTPACK detection engineering content tree, including phase packs, cross-platform rule content, demo tooling, and deployment automation.

## Main branches
- `APTPACK-IA` for Initial Access
- `APTPACK-EXEC` for Execution
- `APTPACK-PE` for Persistence / Privilege Escalation oriented content
- `APTPACK-DISC` for Discovery
- `APTPACK-CA` for Command and Control / Credential Access-adjacent campaign tracking depending on content context
- `APTPACK-STAGE` for Staging / Exfil-related content
- `aptpack-ansible` for deployment and validation automation
- `demo` for operator-driven demonstrations
- top-level XMLs for directly staged Wazuh content

## SOC relevance
This branch is the structured detection engineering pack tree that maps adversary behavior into platform-specific content and validation workflows.
